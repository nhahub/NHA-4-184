Key Performance Indicators

Customer Support RAG Chatbot
Project Type: Depi Graduation Project
Domain: AI-Powered E-Commerce Customer Support
Team Size: 5 Members
1 KPI Definitions

The following Key Performance Indicators are used to measure the success and effectiveness of the chatbot system. Each KPI is tracked using the project's monitoring stack (Prometheus, MLflow, and Streamlit).

K1
Response Time
Performance Metric
Description Time taken from receiving a user query to delivering a response
Metric RESPONSE_TIME Histogram
Tracked By Prometheus
Goal Minimize average response latency
K2
Retrieval Confidence
Quality Metric
Description Relevance score of documents retrieved from the vector database
Metric CONFIDENCE_DISTRIBUTION Histogram
Tracked By Prometheus MLflow
Goal Maximize % of queries with confidence > 0.5
K3
User Feedback Score
Satisfaction Metric
Description Ratio of positive (thumbs up) to negative (thumbs down) user feedback
Metric FEEDBACK_COUNTER (positive / negative)
Tracked By Prometheus Streamlit
Goal Maintain positive feedback rate above 80%
K4
Request Throughput
Reliability Metric
Description Total number of chat requests handled successfully by the system
Metric REQUEST_COUNTER (total / by intent)
Tracked By Prometheus Streamlit
Goal Handle concurrent requests with zero downtime
K5
Pipeline Accuracy
AI Quality Metric
Description Percentage of queries correctly routed through the 5 adaptive pipeline paths (casual, high-confidence, medium-confidence, low-confidence-verified, low-confidence-rejected)
Metric Benchmark test suite with 20 test queries across all 5 paths, results logged via test_adaptive_pipeline.py
Tracked By MLflow
Goal All 5 pipeline paths produce correct routing and appropriate responses
2 Monitoring Stack

The KPIs above are tracked and visualized using a three-layer monitoring architecture integrated into the system.

Prometheus

Real-Time Metrics Collection
  • Request counters by intent type
  • Response time histograms
  • Confidence score distribution
  • Feedback counters (positive/negative)
  • Exposed via /metrics endpoint

MLflow

Experiment Tracking
  • Per-query logging (parameters + metrics)
  • Confidence scores per query
  • Pipeline path tracking
  • Configuration comparison experiments
  • Stored locally in mlruns/

Streamlit

Visual Dashboard
  • 5 KPI summary cards at top
  • Confidence distribution chart
  • Response time trend chart
  • Feedback breakdown chart
  • Recent conversations table
