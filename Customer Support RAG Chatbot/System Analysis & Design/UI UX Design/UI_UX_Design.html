1 Design Overview

The frontend is built using Next.js 16 with React 19 and TypeScript. The UI component library is shadcn/ui (built on Radix UI primitives) with Tailwind CSS v4 for styling. The design follows a modern, minimal chat interface inspired by ChatGPT, with full dark/light theme support.

Design Philosophy: Clean, distraction-free interface that puts the conversation at the centre. The UI prioritises readability, accessibility, and a seamless conversational experience across desktop and mobile devices.

6 Key UI Interactions

InteractionBehaviour
Send MessageUser types in textarea and presses Enter (or clicks Send). Message appears instantly on the right side. Bot response appears on the left after loading.
Voice RecordingUser clicks the mic button. The input border turns red with a pulsing animation. A "Recording..." indicator appears. Clicking the mic again stops recording and sends the audio to the voice endpoint.
Typing IndicatorThree animated dots appear on the left side while the AI is generating a response.
FeedbackEach assistant message displays thumbs up/down buttons. Clicking one submits feedback to the backend and visually highlights the selected option.
Auto-Resize InputThe textarea automatically grows as the user types (up to 150px max height), then becomes scrollable.
Conversation SwitchingClicking a conversation in the sidebar loads its messages. The active conversation is visually highlighted.
New ChatClicking "+ New Chat" clears the message area and shows the Empty State with suggestion cards.
Theme ToggleA moon/sun icon button in the header toggles between dark and light themes. Preference is persisted.
Password VisibilityEye/EyeOff icon toggles password field between hidden and visible text on the login page.
Empty State SuggestionsFour clickable cards (e.g., "How do I return a product?") that auto-submit as a message when clicked.

7 Accessibility and Responsiveness

7.1 Accessibility (a11y)

  • Radix UI Primitives: All 57 UI components are built on Radix UI, which provides WAI-ARIA compliant keyboard navigation, focus management, and screen reader support out of the box.
  • Semantic HTML: Proper use of <form>, <label>, <button>, and <header> elements throughout the application.
  • Keyboard Navigation: Enter to send messages, Shift+Enter for newline, Tab navigation for all interactive elements.
  • Colour Contrast: OKLCH colour system ensures perceptually uniform contrast ratios between text and background in both themes.
  • Focus Indicators: Visible outline ring on focused elements using outline-ring/50 utility.
  • Button Titles: Voice recording button includes title attribute ("Voice message" / "Stop recording") for screen readers.

7.2 Responsive Design

BreakpointBehaviour
Desktop (md+)Two-panel layout: sidebar (conversation list) + main chat area side by side.
Mobile (<md)Sidebar is hidden (hidden md:block). Full-width chat interface. Chat input remains accessible at the bottom.

7.3 Theme Support

  • Dark Mode (Default): Deep navy/charcoal backgrounds with light text. Default theme set via defaultTheme="dark" in ThemeProvider.
  • Light Mode: Clean white backgrounds with dark text. Togglable via the theme switch button.
  • System Preference: enableSystem flag allows automatic detection of user's OS preference.
  • Persistence: Theme preference is stored in localStorage and persisted across sessions.
