1 API Documentation

The backend uses FastAPI, which automatically generates interactive API documentation using the OpenAPI 3.0 specification. No separate API documentation is maintained — the docs are always in sync with the actual codebase.

1.1 Built-in Documentation Endpoints

URLInterfaceDescription
http://localhost:8000/docsSwagger UIInteractive API explorer with "Try it out" functionality. Lists all endpoints, request/response schemas, and authentication requirements.
http://localhost:8000/redocReDocRead-only API documentation with a clean, three-panel layout optimised for reading.
http://localhost:8000/openapi.jsonOpenAPI JSONRaw OpenAPI 3.0 specification file. Can be imported into Postman, Insomnia, or other API tools.

1.2 Documentation Features

  • Auto-Generated Schemas: All Pydantic request/response models (ChatRequest, ChatResponse, SourceChunk, etc.) are automatically documented with field types, descriptions, and validation rules.
  • Authentication: Endpoints requiring JWT authentication are clearly marked with a lock icon in Swagger UI.
  • Tags: Endpoints are grouped by tag — Chat, Auth, Feedback — for easy navigation.
  • Example Values: Pydantic models provide example values for all request fields.

Access: Start the backend server with uvicorn app.main:app --port 8000 and navigate to /docs in your browser to explore the full interactive API documentation.

3 Testing Strategy

Current Status: Unit testing has not been implemented in the current version of the project. The section below outlines the planned testing strategy for future development phases.

3.1 Planned Testing Approach

Test TypeFrameworkScope
Unit TestspytestIndividual functions: Embedder.embed_text(), LLMRouter.classify(), Generator.rewrite_query()
Integration Testspytest + httpxFull API endpoint testing: POST /chat/ask, POST /auth/login, POST /feedback
API Contract TestsFastAPI TestClientValidate request/response schemas match Pydantic models
Database Testspytest + SQLAlchemyORM model CRUD operations with test database

3.2 Current Validation Methods

While formal unit tests are not yet in place, the following validation mechanisms are actively used:

  • Pydantic Validation: All API request bodies are validated at runtime by Pydantic models, rejecting malformed data with clear error messages.
  • FastAPI Swagger UI: Manual endpoint testing via the interactive /docs interface during development.
  • Structured Logging: Comprehensive logging throughout the NLP pipeline (router, generator, retriever) to trace request flow and identify issues.
  • Prometheus Metrics: Real-time tracking of request counts, response times, and retrieval confidence scores.
  • MLflow Tracking: Experiment logging to compare pipeline performance across different configurations.

4 Glossary of Terms

TermDefinition
RAGRetrieval-Augmented Generation — a technique that combines information retrieval with LLM generation to produce grounded, accurate answers.
LLMLarge Language Model — a deep learning model trained on vast text data to understand and generate natural language (e.g., Llama 3.3 70B).
EmbeddingA dense numerical vector representation of text that captures semantic meaning, enabling similarity comparison.
Vector DatabaseA database optimised for storing and querying high-dimensional vectors (e.g., ChromaDB).
Semantic SearchSearch based on meaning rather than keyword matching, using vector similarity to find relevant documents.
HNSWHierarchical Navigable Small World — an approximate nearest neighbour algorithm used by ChromaDB for fast vector search.
STTSpeech-to-Text — converting spoken audio into written text (implemented via OpenAI Whisper).
TTSText-to-Speech — converting written text into spoken audio (implemented via ElevenLabs API).
JWTJSON Web Token — a compact token format used for secure authentication between frontend and backend.
OAuth 2.0An authorisation framework that allows third-party login (Google) without sharing passwords.
OTPOne-Time Password — a temporary code sent via email for user verification during registration.
ASGIAsynchronous Server Gateway Interface — the Python standard for async web servers (used by Uvicorn/FastAPI).
ORMObject-Relational Mapping — a technique for interacting with databases using Python objects instead of raw SQL (SQLAlchemy).
OKLCHA perceptually uniform colour space used in Tailwind CSS v4 for consistent colour contrast across themes.
Adaptive RAGAn enhanced RAG pipeline that uses intent routing, query rewriting, confidence scoring, and answer verification to optimise response quality.
